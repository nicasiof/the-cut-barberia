Especificación de Requisitos Según el estándar de IEEE 8830.

by Nicasio Freytez

Resumen.

Este documento presenta, la Especificación de Requisitos Software (ERS) según la última versión del estándar IEEE 830 para el sitio web de la empresa
The cut barbería social club (el cliente) con sede en la Ave. Nueva Prodivencia 2212 loc 205. El sitio viene a complementar la presencia en medios digitales de el cliente. La que, hasta el momento utiliza instagram como medio para ofrecer sus servicios. Sin embargo, no se muestran las comodidades que ofrece similares a las de un club social familiar. Este sitio web se desarrolla con un propósito netamente educativo. Todos los medios digitales empleados son propiedades de sus productores: el cliente como persona jurídica. El producto es un prototipo parcialmente funcional que mostrará todas las vistas, diseñado para apuntar a dos páginas: tienda y reserva. Por requerimentos didácticos y para generar una experiencia más completa, se incluirán en el diseño otras páginas en la estructura: main, contactos y anular reserva. Las cuales son adicionales a las solicitadas por el cliente.

1. Introducción.

El cliente, es un prestador de servicios varios: barbería, expendio de bebidas con y sin alcohol, venta de productos para el cuidado personal por mayor y detal. Es una empresa con sede única en Chile, Región Metropolitana en la ciudad de Santiago. Ubicada en la Ave. Nueva Providencia 2212 Loc. 205.

El inicio formal de operaciones fue el 5 de julio de 2022. Buscando mostrar los atributos que la diferencian de la competencia El cliente, tiene un propuesta que va más allá de un corte de cabello y barba.

En sus instalaciones y por su espacio y diseño, además de servicio de barbería el local ofrece un espacio de entrenimiento con cónsolas de vídeo juegos y una recepción con bar. Combinado con un espacio donde se muestran y se usan productos de cuidado personal que están a la disposición de la compra de cualquier ineteresado.

Se busca entonces, una manera que nuevos interesados puedan comprender este nuevo tipo de propuesta, empleando un espacio web con este propósito. Donde a diferencia de las redes sociales se podrá tener una vista más controlada de lo que El cliente quiere mostrar


    1.a. Propósito.

    Desarrollar un prototipo no funcional de un sitio web que le permita a los usuarios: reservar horas, interactuar con un catálogo de productos y familiarizarlos con la experiencia que ofrece El cliente Considerando esto se propone un sitio web de cinco páginas para satisfacer las necesidades de El cliente.


    1.b. Ámbito del sistema.

    • El sistema será denominado como "sitio web de The cut barbería social club".
    • El sitio web de The cut barbería social club, mostrará multimedia relacionado con actividades de El cliente.
    • Se desarrollará como un todo, conformado de cinco páginas: main, reservas, tienda, anular reserva y contacto.
    • La página main: mostrará información de interés general.
    • La página reserva: simulará el proceso de agendar una hora.
    • La página tienda: simulará el proceso de comprar un producto.
    • La página contacto: simulará el proceso de enviar un mensaje, mediante el uso de un formulario.

    1.c. Referencias.

    La fuente de la información es El cliente. Se obtuvo mediante la técnica de la entrevista, llevada a cabo a través de un guión de preguntas abiertas que no estaban estructuradas.


2. Descripción general.

Este documento pretende de forma ordenada, el acuerdo de diseño al que han llegado El cliente. y El desarrollador. La intención es de generar acciones que conlleven a la geración de vistas ordenadas que simulen el funcionamiento.
Se aplicarán tecnologías: html, css, framewoks, sass y criterios seo.

    2.a. Propósito.

    Desarrollar un prototipo no funcional de un sitio web que le permita a los usuarios: reservar horas, interactuar con un catálogo de productos y familiarizarlos con la experiencia que ofrece El cliente.
    Considerando esto se propone un sitio web de cinco páginas para satisfacer las necesidades de El cliente y los objetivos que se establecen en la materia.

    2.b. Funciones de producto.

    Entre las funciones del producto se encuentran:

    • Colocar el logo de el cliente en cada página del sitio.
    • Simular de manera fidedigna los procesos de: reserva, anulación de reserva, contacto y compra.
    • Insertar un vínculos a la redes sociales de el cliente
    • Proveer una experiencia cercana al funcionamiento del sistema.

    2.c. Características de los usuarios.

    Los usuarios serán los interesados del sistema. Modificaciones de contenido, forma, funcionamiento, entradas y salidas de datos, se harán por El desarrollador.

    2.d. Restricciones de diseño.

    Por las necesidades de el cliente y las características de diseño se definen las siguientes limitanciones:


    • El sistema no poseerá una base de datos para gestionar: clientes, reservas, personal o productos.
    • El alcance del desarrollo es el prototipado no la funcionalidad.
    • No se establece un anillo de usuarios ya que el prototipo será utilizado para representar el funcionamiento. Es decir, será operable pero no funcional.
    • Las entradas serán preestablecidas y las salidas serán simuladas.
    • No se desarrolla una página de galería de fotos ya que el cliente quiere manejar esos recursos vía instagram.

    2.e. Requisitos futuros.

    Por las necesidades de el cliente y las características de diseño se definen los siguientes requisitos futuros:


    • Diseñar, desarrollar e implantar una base de datos relacional.
    • Adquirir el dominio y un hosting.
    • Desarrolar una página con una galería que muestre al staff con algún tipo de información relevante.
    • Incluir un módulo de usuarios para así facilitar el proceso de reservas y comunicar promociones.

    3.b. Wireframe

    Página de bienvenida.

    Click en el logo ingresa a la página principal.
    Click en el botón de reservas se ingresa para agendar horas.
    Click en el boón de compras se ingresa a la tienda.


    Página principal.
    Página donde se muestra una reseña de la compañía y una imagen que se complementa con una reseña.


    Página de contacto.
    Contiene un formulario que permite el envío de un mensaje breve.

    Página de anular reserva.
    Indica la política de cancelación y muestra el formulario para solicitar la cancelación

    Página de reserva.
    Muestra una foto y el nombre del estilista que lo atenderá.
    Al hacer click en el botón del nombre, se pasa al siguiente paso de la reserva.

    Página de selección de servicio.
    Se selecciona de una lista el servicio a contrar y se muestra su costo.
    Se mantiene a la vista el estilista elegido.

    Página de selección de fecha.
    Se selecciona de un calendario fecha y turno.
    Se mantiene a la vista el estilista elegido y el servicio.

    Página de pago.
    Se verifican el pedido y se ingresan los datos de pago.

    Página de pago exitoso.
    Muestra una pantalla informativa al usuario que el pago fue aprobado. Puede ser sustituída en el futuro por un mensaje.

    Página de pago fallido.
    Muestra una pantalla informativa al usuario que el pago fue aprobado. Puede ser sustituída en el futuro por un mensaje.

    Tienda.
    Arreglo ordenado de cartas que muestra productos con descripción y precio.

    Se verifica el pedido y se ingresan los datos de pago.

4. Conclusión.

Se ha mostrado la lógica de diseño de un sitio web, orientado a generar presencia de un establecimiento comercial en internet. La estructura que se plantea satisface los requerimientos expresados. El diseño de las interfaces y el flujo de las páginas puede y será modificado según se avance en el desarrollo para cumplir con las exigencias de el cliente.
Aunque se hagan modificaciones en las interfaces, siempre se tratará de mantenerse fiel al diseño original y sus componentes.
